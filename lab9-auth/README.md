# Лабораторна робота №9: Аутентифікація в Nest.js

Цей проєкт містить реалізацію системи реєстрації та авторизації користувачів для Nest.js застосунку.

## Технології
- Nest.js
- PostgreSQL
- JWT (JSON Web Tokens) для захисту маршрутів
- bcrypt для безпечного хешування паролів у базі даних

## Встановлення та запуск

1. Встановіть залежності:
npm install

2. Налаштуйте змінні оточення. Створіть файл .env у корені проєкту та додайте ваші дані (приклад):
DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=postgres
DB_PASSWORD=ваш_пароль_від_бд
DB_DATABASE=lab9_auth
JWT_SECRET=ваш_унікальний_секретний_ключ

3. Запустіть сервер у режимі розробки:
npm run start:dev

## Доступні API маршрути

- POST /auth/register
Створення нового користувача. Тіло запиту повинно містити email та password. Пароль автоматично хешується перед збереженням у базу даних.

- POST /auth/login
Аутентифікація користувача. Приймає email та password. У разі успіху повертає access_token (JWT).

- GET /auth/me
Отримання даних профілю поточного користувача. Цей маршрут захищений за допомогою JwtAuthGuard. Для доступу необхідно передати валідний JWT токен у заголовку Authorization (тип Bearer).
